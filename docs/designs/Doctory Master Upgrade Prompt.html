§ 01 · Diagnosis

What's broken right now

Audited against the live codebase at lawrns/doctormx@main. Prioritised by user impact.

UX-01

/ai-consulta — no agentic tool calling, no clinical UX

The current page is a thin wrapper around PreConsultaChat with no visible AI reasoning, no tool-call visualization, no urgency signal, no confidence bar, and no COFEPRIS badge. Users have no idea what the AI is doing. The referral section post-completion renders unstyled cards with dead links. This is the single most important page for conversion and trust.

P0 · Critical
UX-02

/doctors — "Agendar cita" button missing from every card

The DoctorsDirectoryClient.tsx card-right panel renders a date grid and "Ver más" only. There is NO primary booking CTA on the most important page in the funnel. Users selecting a date have nowhere to go. The card click dispatches two competing handlers (onSelect + router.push), causing selection-vs-navigation confusion.

P0 · Critical
UI-03

/doctors — Map pin tooltips never show

The tooltip element has opacity-0 group-hover:opacity-100 but the parent pin button lacks the group Tailwind class, so no tooltip ever appears. Also, pins and cards fall out of sync when the results panel scrolls: pin selection should auto-scroll the corresponding card into view.

P1 · High
UX-04

/doctors — Filter sidebar is purely cosmetic

All checkbox filters, the sort select, and the search bar on the directory page have no effect on the displayed results. The sort select has a duplicate "Calificación" option at value="rating". Specialty filter links do work (server-side via URL params) but their visual state doesn't reflect the active filters properly on load.

P1 · High
UX-05

/doctors — No doctor profile action after card click

Clicking a card body triggers router.push to /doctors/:id, but that page is a full server component that may be slow or show a loading skeleton. There's no preview drawer or sheet, so the user loses their place in the results list. A profile sheet (slide-in from right) would retain context and reduce round-trips.

P1 · High
TR-06

Trust signals invisible on /ai-consulta and /, weak everywhere

COFEPRIS compliance badge is absent from the AI consultation page entirely. The doctor cédula badges on cards are not clickable links to SEP verification. Every AI-generated message lacks a "Asistente IA · No es doctor" disclosure tag. The prescription generation flow has an indeterminate spinner with no phase labels (Redactando → Validando COFEPRIS → Firmando).

P1 · High
MOB-07

Patient portal has no mobile navigation

The patient dashboard layout hides the sidebar below lg breakpoint with no hamburger, no bottom-nav, and no sheet fallback. 70%+ of Mexican medical traffic is mobile. This locks the majority of patients out of their own appointment history, prescriptions, and messages.

P1 · High
UX-08

8 empty states show "No hay datos" — all are missed moments

Patient dashboard, messages, prescriptions, consultation history, search results, doctor queue, payment failure, and AI low-confidence all render generic placeholder copy. Every empty state is a critical decision point where the user decides whether to stay or churn. They all need action + warmth.

P2 · Important
PERF-09

Doctor "En revisión" limbo — 8 padlocks, no queue position

New doctors completing onboarding see a sidebar with 8 padlock icons and "24-48 horas hábiles". No queue position, no estimated time, no productive checklist. This destroys doctor-side activation. Replace with a productive waiting room: queue position, tech check, first-consult playbook.

P2 · Important

§ 03 · /ai-consulta — agentic design spec

What "truly agentic" looks like

The AI pre-triage page must show its work. Every tool call is a trust signal. Every confidence bar is a conversion moment.

Layout

Centered chat · max-w 720px

Welcome state with symptom chips. Active state with chat + collapsible tool sidebar. Completion state with doctor referral cards inline.

Tool calls

Visualize every reasoning step

analyze_symptoms → assess_urgency → match_specialty → find_doctors. Each appears sequentially with a spinner, then transitions to ✓ done with output text.

After each AI message

Urgency badge + confidence bar

Color-coded: leaf=low, amber=moderate, rust=high, crimson=emergency. Confidence 0–100 gradient bar. "Asistente IA · No es médico" disclosure mandatory.

Trust (always visible)

COFEPRIS strip + encryption

Sticky header with COFEPRIS badge. "Datos cifrados extremo a extremo". "En caso de emergencia llama al 911." Never dismissible during triage.

Doctor referral

Inline after readyForReferral

3 doctor cards with specialty match %, cédula badge, next available slot, price, and a primary "Agendar cita" CTA. Booking modal — no page reload.

Escalation

"Hablar con doctor humano" always

Ghost button always visible. After 3 AI turns → proactive suggestion. Low-confidence AI response → immediate escalate CTA. Video fallback ladder: video → audio → SMS.

Claude system prompt for /ai-consulta (haiku-4-5 · 1024 token cap)
You are Dr. IA, the AI pre-triage assistant for Doctor.mx, Mexico's leading telemedicine platform.
CRITICAL: Respond ONLY with valid JSON. No other text. Exact structure:
{"toolCalls":[{"tool":"analyze_symptoms","output":"finding"},{"tool":"assess_urgency","output":"assessment"},{"tool":"match_specialty","output":"specialty + confidence"}],"message":"Spanish response 2-3 sentences","urgency":"low|moderate|high|emergency","specialty":"Primary specialty","confidence":75,"readyForReferral":false,"followUp":"question if needed"}
RULES: Spanish. Warm + professional. No diagnoses. Set readyForReferral:true after 2nd patient message. Keep messages under 60 words.
§ 04 · /doctors — pixel-perfect spec

Every broken thing, fixed precisely

The directory is 90% there. These 6 surgical fixes take it to production-ready.

Fix 1 — Missing primary CTA

Add "Agendar cita" button to every card-right

Below the date grid, add a full-width primary button: bg-ink text-white font-bold rounded-lg py-2. Clicking → opens SlotGrid sheet (not navigate away). Date selection in the grid should pre-populate the sheet.

Fix 2 — Map pin tooltips

Add `group` class to pin button

Change: <button className="..."> → <button className="group ...">. The tooltip already has group-hover:opacity-100 — it just needs the parent marker. Also add z-20 so tooltip isn't clipped by the map container.

Fix 3 — Card click conflict

Separate select vs. navigate intents

Card click → onSelect(idx) (highlights + syncs map). "Ver perfil" button inside → router.push(). Never do both on the same click. Add a dedicated "Ver perfil ›" ghost button in card-left footer.

Fix 4 — Sort select

Remove duplicate option, wire onChange

The sort select has two value="rating" options. Remove one. Wire onChange to update router.push with sortBy and sortOrder query params — the server component already reads them.

Fix 5 — Profile drawer

Sheet slides in from right on card click

Use shadcn <Sheet side="right">. Content: avatar + bio + reviews + map embed + SlotGrid + Stripe booking button. Retains results list context. Closes on Escape or overlay click.

Fix 6 — Overflow on small viewports

Clamp the 3-column layout

Change grid-template-columns: 220px 1fr 380px to minmax(180px, 220px) 1fr minmax(320px, 380px). Map panel: min-width: 0. Results panel: overflow-y: auto; min-width: 0. Sidebar: overflow-y: auto; min-width: 0.
